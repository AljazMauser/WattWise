# WattWise

WattWise is a Cycling Training Tracker & AI Coach web application. It allows users to upload cycling data files (.FIT and .GPX), visualize their rides with interactive maps and charts, and receive human-like, analytical training commentary generated by an LLM based on their exact ride metrics.

## Features

- **User Authentication**: Secure registration and login.
- **File Upload & Processing**: Support for `.GPX` (XML) and `.FIT` (binary) uploads. Extracts timestamps, coordinates, altitude, speed, heart rate, cadence, and power.
- **Interactive UI**: View route maps using Leaflet and detailed metric charts using Recharts.
- **Cycling Analytics Engine**: Calculates Distance, Time, Average Speed, Max Speed, Elevation Gain, Average HR, Max HR, and Average Power.
- **Advanced Power Metrics**: Calculates Normalized Power (NP), Intensity Factor (IF), and Training Stress Score (TSS).
- **AI "Ride Replay" Coach**: Generates a personalized coach review of your ride using the Groq API (Llama 3).

## Tech Stack

- **Frontend**: Next.js (App Router, TypeScript), Tailwind CSS, Recharts, React-Leaflet
- **Backend**: Python (FastAPI), SQLAlchemy (Async), fitparse, pandas
- **Database**: PostgreSQL
- **AI Integration**: Groq Cloud API (llama3-8b-8192)
- **Infrastructure**: Docker Compose

## Setup Instructions

1. **Clone the repository**:
   ```bash
   git clone https://github.com/AljazMauser/WattWise.git
   cd WattWise
   ```

2. **Environment Variables**:
   Copy the example environment file and fill in your details (importantly, your Groq API key).
   ```bash
   cp .env.example .env
   ```

3. **Run with Docker Compose**:
   ```bash
   docker-compose up --build
   ```

4. **Access the application**:
   - Frontend: `http://localhost:3000`
   - Backend API Docs: `http://localhost:8000/docs`

## Development

- Frontend is located in the `/frontend` directory.
- Backend is located in the `/backend` directory.
- Database migrations are handled by Alembic in the `/backend/alembic` directory.
